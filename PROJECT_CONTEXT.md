## 任务：开发“失意者联盟”多页面叙事 Web 应用

### 1. 项目概括

构建一个基于 eFootball 截图识别的模拟叙事网站。系统需支持多轮、不同场次的截图上传，并通过 Vision API 驱动球员人设演变与动态舆论生成。

### 2. 核心架构与页面逻辑 (3-Page System)

#### **第一页：球队组建 (Recruitment Page)**

- **交互逻辑**：
  - 玩家上传一张“球队计划/阵型截图”（例如包含首发名单的页面）。
  - **AI 任务**：识别所有球员的姓名、位置、评分。
  - **叙事注入**：为识别出的每个球员生成一个**“失意者背景”**（例：过气天才、更衣室炸弹、长期伤号）。
  - **持久化**：将这些数据（球员列表+人设）存储在 `localStorage` 中作为项目初始状态。

#### **第二页：动态媒体中心 (Media & Forum Center)**

- **交互逻辑**：
  - 玩家在完成每一场 eFootball 比赛后，上传 1-3 张赛后统计截图（评分、控球率、进攻区域）。
  - **AI 任务**：
    1. **数据解析**：对比 `localStorage` 中的球员列表，提取本场比赛的表现评分（Rating）。
    2. **生成新闻**：根据比分和数据，生成一篇主流体育媒体风格的简报（如《踢球者》风）。
    3. **生成论坛回帖**：在新闻下方生成 5-10 条 BBS 风格回帖。
       - **风格限制**：必须包含现代足球梗（“牢X”、“神仙球反讽”、“依托构思”、“顶级理解”）。
       - **逻辑驱动**：若球员连续表现不佳，回帖 Hostility（敌对度）增加；若逆袭，出现“道歉信”类回帖。

#### **第三页：主教练面板 (Manager Dashboard)**

- **视觉展示**：
  - **人设卡片**：列出所有球员及其动态更新的“救赎值”。
  - **舆论概览**：用折线图或进度条展示球迷对球队的实时印象分。
  - **历史记录**：展示过去几场比赛的简短评价摘要。

### 3. 技术实现细节 (Technical Requirements)

- **API 集成**：使用豆包 Vision API (`doubao-seed-1-8-251228`)。

- **状态管理**：

  JavaScript

  ```
  // 建议的数据结构
  const playerState = {
    id: "player_name",
    backstory: "...",
    redemptionScore: 50, // 初始值
    matchHistory: [{ rating: 8.0, sentiment: "hype" }]
  };
  ```

- **视觉风格**：

  - **新闻区**：干净的现代网页版面，白色背景。
  - **论坛区**：2000 年代 BBS 风格（#111 背景，绿色点阵字，粗糙的 1px 边框）。

### 4. 关键提示词（请 AI 在生成文本时使用）

> “你现在是一个混合身份：一半是毒舌足球解说，一半是资深论坛键盘侠。请根据传入的球员评分，结合他们‘失意者’的人设，生成带有强烈偏见和分裂感的文字内容。”
